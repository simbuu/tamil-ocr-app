Reading guide. The system is a single FastAPI service. A request flows browser → page route (Jinja2) for HTML, or browser → /api/* router → service → SQLAlchemy model → SQLite for data. The OCR critical path (diagram 2) is the only complex one: an upload triggers a preprocessing + EasyOCR + row-parsing pipeline, the raw output and timings are persisted as an OCRSession, the parsed rows are price-enriched and previewed, and on confirm each row is diffed against the original OCR output so per-field edit counts can be saved on the Transaction. That edit metadata is what the analytics_service later turns into accuracy reports.
